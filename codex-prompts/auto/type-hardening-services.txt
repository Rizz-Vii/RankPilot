# Codex Autonomous Prompt: Type Hardening (Service + Server Visualization Layer)

Objective:
Systematically eliminate unsafe `unknown` usages, broad casts (`as unknown`), and implicit any patterns across the remaining service layer (marketing, sales, team) and server visualization export modules (`server-exports.ts`, `server-artifacts.ts`). Produce strongly typed domain contracts without inflating complexity or introducing derived data persistence. Keep each batch minimal, deterministic, and under LOC limits.

Strategic Goals:
1. Replace all `unknown` spreads `(d.data() as unknown)` in services with typed snapshot mappers.
2. Introduce shared domain types (Firestore DTOs + mapped runtime shapes) in a single `types/firestore-docs.ts` (create if absent) or reuse an existing central types file if more appropriate after scan.
3. Refactor `server-exports.ts` and `server-artifacts.ts`:
   - Remove `format as unknown` casts.
   - Add discriminated union for formats.
   - Introduce `ServerChartArtifactData` and `ServerDashboardArtifactData` minimal shapes.
   - Ensure no property access on `unknown` remains (zero TS18046 errors locally after refactor).
4. Team service: define `RawTeamMember` (Firestore shape) vs `TeamMember` (runtime). Replace `mapMember(raw: unknown)` signature with `mapMember(raw: RawTeamMember)` eliminating unsafe optional chaining on unknown. Provide minimal field guards.
5. Add generic snapshot helper: `mapDocs<T>(snap: QuerySnapshot<DocumentData>, mapper: (id: string, data: DocumentData) => T): T[]` (or narrower) to cut repetition.
6. Add lightweight unit contract tests (TypeScript-only or mocha) under `testing/unit/types/` verifying transformations compile and a couple of runtime mapping assertions (e.g., campaign aggregation, server artifact format guard). Keep tests <120 LOC total.
7. Ensure no derived ratios (CTR, ROI) get persisted—only computed in runtime aggregation; maintain existing behavior.
8. Preserve existing public exports & function names (non-breaking) unless renaming purely internal helpers (ok) — document renamed internals inline as comments.
9. Keep diffs segmented: one batch per logical cluster (services marketing + helper, then sales, then team, then server visualization export, then tests). Each batch < 160 LOC added (hard stop 200). Abort batch if risk of exceeding.
10. After each batch: run `npm run typecheck` and if available `npm run test:unit:types`. If failures: self-correct within same batch (preferred) OR open a follow-up batch with narrow scope fix.

Acceptance Criteria per Batch:
- TypeScript: 0 new errors introduced; net reduction of `unknown` property access warnings.
- No introduction of `any` unless explicitly justified in inline comment referencing reason + TODO conversion path.
- Firestore DTOs expose only raw stored fields: do not add derived metrics or ratios.
- Runtime mapped shapes use concrete types (Date, number, string) without `unknown`.
- Server artifact generation uses explicit param interfaces; no implicit `any` in exported functions.
- Tests (if touched) pass locally.
